## 데이터 모델링
### 개념
- 현실 세계의 데이터를 데이터베이스로 옮기기 위해 구조화하는 작업
```
  추상화 : 현실세계를 데이터로 표현할 때 중요한 요소만 선택해서 표현한다
  단순화 : 누구나 이해할 수 있도록 쉽게 표현해야 한다
  명확화 : 의미가 모호하지 않도록, 오해없이 해석되게 만든다
```

## DML
### 1. SELECT (조회)
``` sql
  SELECT 컬럼명1, 컬럼명2, ...
  FROM 테이블명
  WHERE 조건식;
```
#### NULL 비교
``` sql
  -- 컬럼 값이 NULL인 행만
  WHERE 컬럼명 IS NULL;
  -- 컬럼 값이 NULL이 아닌 행만
  WHERE 컬럼명 IS NOT NULL;
```
#### SQL 연산자
BETWEEN : A와 B 사이의 값을 조회할 때 사용  
IN : OR을 대신해서 사용하는 연산자
``` sql
  WHERE 컬럼명 BETWEEN '비교값1' AND '비교값2';
  WHERE 컬럼명 in('비교값1', '비교값2', ...);
```
#### LIKE 연산자
% : 0글자 이상 아무 문자  
_ : 딱 1글자 아무 문자  

##### 예시 패턴
'M%' : M으로 시작  
'%a' : a로 끝남  
'%a%' : a를 포함
'M_____' : M으로 시작 + 총 6글자

### 2. INSERT(추가)
테이블에 새로운 행을 추가할 때 사용하는 DML이다.
```
  INSERT INTO 테이블명 (컬럼1, 컬럼2, 컬럼3)
  VALUES (값1, 값2, 값3);
```
### 3. UPDATE(수정)
데이터를 수정할 때 사용하는 DML이다.
``` sql
  UPDATE 테이블명
  SET 컬럼1 = 값1,
      컬럼2 = 값2
  WHERE 조건식;
```
※ UPDATE에서 WHERE을 잊으면 안된다.

### 4. DELETE(삭제)
테이블 안의 데이터를 삭제할 때 사용하는 DML이다
``` sql
  DELETE FROM 테이블명
  WHERE 조건식;
```
