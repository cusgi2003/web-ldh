## DBMS(Database MAnagement System)
  ```
    데이터 : 값을 나타내는 것 ( 숫자, 문자, 이미지, 비디오, 날짜)
    데이터베이스 : 데이터를 활용하기 위해 통합하여 저장한 운영 데이터의 집합
  ```
  
  - 데이터베이스와 사용자 또는 프로그램 간의 매개체 역할
  - 사용자가 정보의 구성, 검색, 수정, 삭제와 같은 관리를 할 수 있게 해준다

  ### DBMS의 종류
    1. 계층형 데이터베이스
      - 폴더와 파일 등의 계층 구조로 데이터를 저장하는 방식
      - 데이터의 관계를 트리 구조로 정의, 부모-자식 형태를 가짐
      - 데이터 액세스 속도가 빠름
      - 데이터의 사용량 쉽게 예측
      - 상하 종속적인 관계로 구성되어 초그 세팅 후 프로세스 수용이 어려움
       
    2. 네트워크형 데이터베이스
      - 데이터 구조를 네트워크상의 노드 형태로 논리적이게 표현한 데이터 모델, 각각의 노드를 서로 대등한관계로 구성한 시스템
      - 데이터의 중복 문제 해결, 상하 종속적 관계 해결
      - 추후 변경시 복잡합 구조로 인해 변경이 어려움
      - 데이터 종속성을 해결하지 못함
    
    3. 관계형 데이터베이스
      - 행과 열을 가지는 표 형식 데이터를 저장하는 형태의 데이터베이스

  ### 모델링
  테이블을 만들기 전에 요구사항을 잘 분석하고 무결성을 지키기 위한 계획
  ```
    개념적모델링 - 엔티티구성, ERD 그리기  
    논리적모델링 - 정규화, 속성 자료형, 제약조건 설정  
    물리모델링 - 실제로 테이블을 만드는 단계
  ```

  ### 정규화
    이상 현상을 제거하기 위해 테이블을 분리하는 것

      삽입이상 : 데이터를 추가할 때 필요없는 내용까지 추가되는 것  
      갱신이상 : 중복된 데이터가 있을 때 일부분만 수정하는 것
      삭제이상 : 데이터를 삭제할 때 필요한 내용까지 삭제되는 것

      제 1차 정규화 : 모든 속성을 원자값이어야 한다
      제 2차 정규화 : 부분함수 종속을 해소한다
      제 3차 정규화 : 이행함수 종속을 해소한다
      BCNF : 3차 정규화를 좀 더 엄격하게 관리한다 ( 결정자는 반드시 후보키여야 한다)
      제 4차 정규화 : 관계없는 속성은 분리한다
      제 5차 정규화 : 조인 종속을 제거한다


    
## 스키마(데이터베이스)
- 데이터베이스의 구조, 테이블, 뷰, 제약 조건 등과 같은 메타데이터를 정의한 것
- 데이터를 저장하고 관리하기 위한 전체적인 설계도 또는 컨테이너
- 스키마는 테이블의 열(컬럼)에 어떤 종류의 데이터가 들어갈지, 데이터의 형식과 제약 조건은 무엇인지 등 데이터의 구조를 정의
```
  외부 스키마 : 사용자가 데이터베이스를 어떻게 인식하는지에 대한 뷰(view)를 정의
  개념 스키마 : 데이터베이스 전체 구조를 논리적으로 정의, 데이터 간의 관계와 보안등을 포함
  내부 스키마 : 데이터가 물리적 저장 장치에 어떻게 저장되는지를 정
```
### SQL문
  - 구조화된 질의언어(Structuerd Query  Language)의 약자로 DBMS 상에서 데이터를 읽고 쓰고 삭제하는 등
    관리하기 위한 일종의 프로그램 언어
    ```
      1. DDL(Data Definition Language) : 데이터 정의어
        - 데이터 베이스 객체를 관리하는 언어
      2. DML(Data Manipulation Language) : 데이터 조작어
        - 데이터 삽입, 조회, 삭제, 갱신 등 실제 데이터를 조작하는 언어
      3. DCL(Data Controll Language) : 데이터 제어어
        - grant, revoke 등 키워드를 가지는 문장
      4. TCL(Transaction Controll Language) : 트랜잭션 언어
        - commit, rollback, savepoint
    ```
#### 데이터베이스 생성
```
  CREATE DATABASE IF NOT EXISTS '데이터베이스 이름'
  CHARACTER SET = utf8mb4
  COLLATE = utf8mb4_unicode_ci
```
▷ IF NOT EXISTS : 같은 이름의 데이터베이스가 존재하면 오류내지 않고 그냥 무시한다.  
▷ CHARACTER SET = utf8mb4 : 문자를 저장할 기본 문자셋을 지정한다.  
▷COLLATE = utf8mb4_unicode_ci : 정렬과 비교의 기준을 설정한다.

#### 행
- 테이블을 구성하는 데이터들 중 가로로 묶은 데이터셋
- 관계형 데이터베이스에서는 튜플(tuple), 또는 레코드(record)라는 이름으로 불린다.
- 행의 수를 기수(Cardinality)라고 한다.
#### 열
- 테이블을 구성하는 데이터들 중 세로로 묶은 데이터셋
- 열을 구성하는 값들은 도메인으로 되어있다.
- 관계형 데이터베이스에서는 속성(attribute)라는 이름으로 불린다.
- 열의 수를 치수(Degree)라고 한다.

#### 도메인
- 하나의 속성이 취할 수 있는 동일한 유형의 데이터 집합
### 제약조건
- 테이블에 문제가 되는 결함이 있는 데이터가 입력되지 않도록 미리 지정해둔 조건
- 테이블 생성할 때 함께 설정할 수 있고, 추후에 생성하거나 변경할 수도 있다.

#### NOT NULL
```
  컬럼명 데이터타입 NOT NULL
```
컬럼을 정의할 때 NOT NULL 제약 조건을 명시하면 해당 컬럼에는 반드시 데이터를 입력해야 한다  

#### UNIQUE
```
  컬럼명 데이터타입 UNIQUE;
  또는
  CONSTRAINTS 제약조건명 UNIQUE(컬럼명, ...);
```
해당 컬럼에 들어가는 값이 유일해야 한다는 의미이다.  
즉, 중복되는 값을 허용하지 않는다.

#### PRIMARY KEY(기본키)
기본키는 UNIQUE와 NOT NULL속성을 동시에 가진 제약조건으로,  
테이블당 1개의 기본키만 생성할 수 있다
```
  컬럼명 데이터타입 PRIMARY KEY;
  또는
  CONSTRAINTS 제약조건명 PRIMARY KEY(컬럼명, ...);
```

#### FOREIGN KEY(외래키)
```
  FOREIGN KEY(컬럼명) REFERENCES 참조테이블(컬럼명);
```
##### 외래키 제약사항
```
  반드시 참조하는 테이블이 먼저 생성되어야 하며, 참조키가 참조 테이블의 기본키로
  만들어져 있어야 한다
  여러 컬럼을 외래키로 만들려면, 참조하는 컬럼과 이래키 컬럼의 순서와 개수는 같아야 한다
```
