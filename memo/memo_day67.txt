JavaScript
화면을 동적으로 움직일 수 있게 해주는 프로그래밍 언어

할 수 있는 것
화면의 조작 : HTML 요소를 선택하고, 읽고 바꾸기
이벤트 처리 : 클릭, 입력, 스크롤, hover 등이 발생했을 때 처리
데이터 처리 : 연산, 조건, 반복
서버와 통신 : API 요청으로 데이터 주고받기

JS로 요청할 때 
ajax ->다양한 객체와 메서드에 대해서 알아야 함(복잡)
fetch() -> api 요청 내장 함수 (상대적으로 간편함)

html에서 js를 사용하는 방법
1. <script> ~ <script> 태그 내에서 작성하기
2. js 파일을 만들어서 작성한 다음 html 내부에서 import 하기
<script src="파일의 경로"></script>
3. 태그 내부에서 이벤트 속성을 통해서 작성하기
<button onclick="눌렀을 때 뭐할건지"></button>

document객체
- 현재 웹 페이지 전체를 의미하는 객체이다.

<html>
	<head>
		<title>제목</title>
	</head>

	<body>
		<h1>안녕하세요</h1>
		<p>JavaScript 공부중</p>
	</body>

</html>

브라우저는 저 코드를 화면에만 그리는게 아니라 객체로도 만들어둔다.

document객체가 필요한 이유
- JavaScript는 HTML을 직접 수정할 수 없다
- 반드시 document를 통해 접근을 해야한다.

<h1 id="title">안녕하세요</h1>

- CSS 선택자로 태그를 선택하는 방법도 있음

document객체가 하는 일
- HTML 요소 찾기
- 글자 변경
- 이미지 변경
- CSS 변경
- 새로운 태그 생성
- 태그 삭제
등을 할 수 있다.

document의 대표적인 메서드
document.write() -> 문서에 글자를 출력한다.
※ body에 있는 내용은 지우고 출력을 하기 때문에 실무에서 많이 쓰지 않는다.

document.createElement() -> 새로운 태그를 만든다.(만드는 역할만 함)

document.body -> 웹페이지의 body에 접근을 한다.

body.appendChild(요소); -> 부모 요소의 마지막 자식으로 새로운 요소를 추가하겠다.

꼭 body가 아니라 내가 js로 선택한 태그의 자식으로도 추가할 수 있다.

변수(varible)
변수 선언을 할 때 var, let, const와 같은 키워드를 사용한다.

var : 예전에 사용하던 변수 선언 키워드
- 같은 변수명으로 재선언도 가능하다.
- 대입이 가능하다.
- 같은 이름의 변수를 실수로 여러번 만들어도 오류가 발생하지 않아서 버그가 생기기 쉽다.

let : 값이 변경될 수 있는 변수를 선언하는 키워드
let 변수명 = 값;
- 대입은 가능하다.
- 같은 변수명으로 재선언하는것은 불가능하다.

const : 값이 변경되지 않는 상수를 선언하는 키워드
- 상수를 선언할 때 값을 초기화 해야한다
- 대입을 할 수 없다
- 같은 이름으로 재선언을 할 수 없다.
- 변하면 안되는 값을 저장할 때 사용한다.

명시적 형변환
함수를 이용한 형변환
java에는 메서드만 존재한다. -> 독립적으로 호출 불가능
js는 독립적으로 호출 가능한 함수가 존재한다.
docuemnt.write() -> write()는 메서드
Number() -> 독립적으로 호출하니까 함수

Number() -> 정수로 형변환
String() -> 문자열로 형변환
Boolean() -> 논리형으로 형변환

암묵적 형변환

'5' - 3 -> 2 숫자로 바뀌어서 연산
