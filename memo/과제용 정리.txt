
1. 변수의 종류


• 기본형 타입 (Primitive Type)

	기본형 타입에는 문자형, 논리형, 정수형, 실수형이 있다.


• 참조형 타입 (Reference Type)

	기본형을 제외한 모든 것이 참조형 데이터 타입이다.


2.  4가지 변수의 종류



2.1 인스턴스 변수 (Non-Static Fields)

• static 키워드 없이 선언된 필드이다.

• 각 객체마다 고유한 값을 가진다.

• 예 : private int currentSpeed;


2.2 클래스 변수 (Static Fields)

• static 키워드로 선언된 필드이다.

• 클래스당 하나의 복사본만 존재한다.

• 모든 인스턴스가 공유한다.

• 예 : public static int numberOfGears = 10;


2.3 로컬 변수 (Local Variables)

• 메서드 내에서 선언된 변수이다.

• 해당 메서드 내에서만 접근 가능하다.

• 예 : void calculateSpeed(int distance, int time) { ... }


2.4 매개변수 (Parameters)

• 메서드나 생성자의 입력으로 받는 값이다.

• 예 : public void setSpeed(int speed) { ... }





 기본 자료형 vs 참조 자료형



1) 기본 자료형 (Primitive Data Type)



💡 데이터를 저장하기 위해 사용되는 자료형이며 변수에 실제 값을 가진다.



2) 참조 자료형 (Reference Data Type)



💡 실제 값을 가지는 것이 아니라 데이터가 저장된 메모리의 주소 값을 가지는 자료형이다.

해당 값은 객체를 참조하는 변수 타입이다.



대표적으로 다음과 같은 타입이 있다.

• String

• Array

• Map, Set

• 클래스(Class), 인터페이스(Interface), 열거형(Enum)



또한 필요에 따라 사용자가 직접 참조형 타입을 정의할 수 있다.



 메모리 구조



1. 메소드 영역 (Method Area)

• JVM이 실행되면서 생성되는 공간이다.

• 프로그램 실행에 필요한 자바 코드 정보를 포함한다.

• 클래스 정보

• 전역 변수

• 상수 필드

• static 필드

• 메서드 코드

• 생성자 코드


2. 힙 영역 (Heap Area)

• new 연산자로 객체 또는 배열을 생성할 때 데이터가 저장되는 공간이다.

• 객체와 배열 등 참조 타입의 실제 데이터가 저장되는 영역이다.

• Garbage Collector가 처리하지 않는 한 데이터는 소멸되지 않는다.

• 스택 영역의 참조 변수는 힙 영역의 메모리 주소를 참조한다.


3. 스택 영역 (Stack Area)

• 지역 변수와 매개변수처럼 일시적으로 사용하는 데이터가 저장되는 영역이다.

• 메서드 호출 시마다 메서드 프레임이 생성되어 저장된다.

• 메서드 호출이 끝나면 프레임은 자동으로 제거된다.

• 프레임 내부에 로컬 변수 스택이 존재한다.

• 기본 타입 변수와 참조 타입 변수가 생성되고 제거되는 영역이다.
