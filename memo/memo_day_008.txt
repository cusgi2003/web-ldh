패키지

	단순히 클래스를 분리하는 용도가 아니라 클래스별 구분자 역할도 한다
	패키지도 계층구조로 만들 수 있다

	com
		ㄴ kumho
		ㄴ hankook

	보통은 회사 도메인의 역순으로 패키지 이름을 정한다
	도트(.)을 기준으로 계층이 생긴다
	
	com.kumho.Tire


클래스를 만들면서 다른 클래스를  필드로 사용할 수 있다

	Car {
		Tire tire;
		Engine engine;
	}

싱글톤 패턴

	객체를 메모리에 한 번만 올리고, 올라간 객체를 계속 사용하는 방식

		1. 생성자를 private으로 정의해서 외부에서 호출하지 못하도록 한다
		2. 클래스 내부에서 static으로 객체를 하나 생성한다
		3. 클래스 내부에서 만든 객체를 사용할 수 있도록 public으로 된 메서드를 만든다


상속

	부모 클래스의 필드나 메서드를 자식 클래스에서 사용할 수 있는 것
	자식 클래스가 상속받을 클래스를 고른다	


오버로딩 ( 메서드의 중복 정의 )
오버라이딩( 메서드의 재정의)
