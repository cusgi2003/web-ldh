운영체제

	하드웨어와 소프트웨어 기능을 사용자가 편리하게 사용할 수 있도록 돕는
	'시스템 소프트웨어' 이다.

	자원(cpu가 해야하는 작업, 메모리의 양 등)을 효율적으로 배분하는
	'총괄 관리자' 역할을 한다.

	목적에 따라 운영체제의 종류와 버전을 선택할지 신중히
	고민해야 한다.

	서버 개발할 때 Linux, 가정용 PC는 Windows
	애플 관련 앱 만들고싶으면 MacOS


관리자 권한

	운영체제 전체를 제어할 수 있는 권한
	시스템 설정을 변경할 수 있다

	윈도우 기준 : 우클릭 -> 관리자 권한으로 실행
		특정 폴더에 접근

	관리자 권한이 없어서 설치할 수 없는 것들을 설치할 수 있다
	
일반 권한

	개인 작업만 가능
	시스템에 영향을 주는 작업은 제한됨


메모리 누수

	사용이 끝난 메모리를 헤제하지 못하고 계속 잡고있는 상태

	Ex) 컵에 물을 따라서 마시는데 남아있는데
	남아있는 컵에 물을 또 따름 -> 결국 넘침


C언어 프로그램 -> 개발자가 직접 메모리 할당과 회수 가능
	어렵긴 하지만 효율적으로 프로그램을 만들 수 있다


GC(가비지 컬렉터)
Java -> JVM 내부에 GC가 있음 -> 객체가 사용이 안되는거 같음 
-> 메모리를 회수함


CPU는 한 번에 하나의 작업만 실행한다
	엄청 빠르게 번갈아 실행하기 때문에 동시에 실행되는 것처럼 보임

	코어
	ㄴ CPU 안에 있는 실제 계산하는 장치
	ㄴ 하나의 코어 -> 하나의 "작업 처리 유닛"

	스레드
	ㄴ 프로그램 안에서 실행되는 작업 흐름
	ㄴ CPU가 실제로 처리하는 단위

	로그
	ㄴ 프로그램이 실행되면서 남기는 기록
	ㄴ 문제 해결의 가장 중요한 단서


운영체제 주요 기능
	
	1. 처리 능력의 향상
		ㄴ 주어진 시간 내 얼마나 많은 일을 처리하는지
	2. 응답시간의 단축
		ㄴ 사용자가 요청한 작업에 대해 결과가 나올때까지 걸리는 시간
	3. 신뢰도 향상
		ㄴ 시스템이 오류없이 안정적으로 동작하고, 
		ㄴ 문제가 생겨도 잘 복구되도록 하는 것
	4. 인터페이스 제공
		ㄴ 사용자와 컴퓨터가 소통할 수 있게 해주는 접점
	5. 입출력 관리
		ㄴ 입력 장치와 출력 장치가 정상적으로 동작하도록 한다
	6. 프로그램 및 사용자 보호
		ㄴ  프로그램끼리 서로 함부로 침범하지 못하게 하고, 
		ㄴ 사용자 데이터도 안전하게 지키는 기능
	7. 주기억 장치(RAM) 관리
		ㄴ 필요한 프로그래에 메모리를 할당하고, 사용이 끝나면 회수한다
	8. CPU 관리
		ㄴ 운영체제가 CPU를 어떤 프로그램에 먼저 사용할 지 결정
	9. 파일 관리
		ㄴ 파일을 만들고, 저장하고, 삭제하고 찾고, 보호하는 기능
	10. 명령어 해석  및 실행
		ㄴ 사용자가 입력한 명령어를 해석하고 실행하는 기능
	

오픈소스

	소스코드가 공개되어있는 소프트웨어
	누구나 자유롭게 열람, 수정, 배포할 수 있다

오프소스 사용하면 좋은 이유
	
	1. 비용 절감
		ㄴ 대부분 무료, 기업에서도 적극적으로 활용
	2. 빠른 개발
		ㄴ 로그인 기능 직접 구현하기 보다 라이브러리를 쓰는게 좋다
		ㄴ - > 개발 속도가 폭발적으로 증가
	3. 검증된 코드
		ㄴ 많은 개발자가 사용
		ㄴ 버그가 빠르게 발견
		ㄴ 안정성이 높다
	4. 커뮤니티 지원
		ㄴ 검색하면 웬만하면 해결방법이 있음
		ㄴ 문서, 예제 풍부하다


라이브러리
	ㄴ 클래스나 인터페이스의 묶음
		ㄴ 필드, 메서드 
	ㄴ 다른 프로젝트에서 라이브러리를 import하여 필드나 메서드를 사용할 수 있다


임베디드
	ㄴ 특정 기능을 수행하기 위해 기기 안에 내장된 컴퓨터 시스템
		Ex) 전자레인지, 세탁기, 자동차, 티비, 에어컨 등등


리눅스를 운영체제라기 보다는 커널이라고 부름
	오픈 소스기 때문에 다양한 배포판이 존재한다
		Ubuntu -> 입문용
		CentOS -> 서버용


클라우드( 전문 회사의 데이터센터의 컴퓨터를 빌리는 방법)

AWS (아마존 웹 서비스) : 설정 ㄴ ㅓ무 복잡합
railWay : 아주 편함


식당에 가면 서빙을 하는 사람 Server
주문(요청) -> 서버 -> 주방 -> 서버 ->고객(응답)
요청 : request
응답 : response

서버(server) : 백엔드 프로그램
요청(request)을 받아서 처리하고 결과(response)를 돌려주는 시스템


자동 설정된 파라미터
	OS가 같아서 설정해둔 기본값
	기본값이 내 서버 환경에는  맞지않을 수 있기 때문에 조심해야 함


개발환경에 필요한 운영체제를 식별하는 방법
	
	1. 정보자산의 분석
	※ 정보자산
		ㄴ 데이터(DB 고객정보, 주문내역, 결제정보)
		ㄴ 소프트웨어(프론트, 백엔드, 운영 프로그램)
		ㄴ 서버컴퓨터, 클라우드, 네트워크 장비
		ㄴ 설계서, API 문서, 개발 메뉴얼, 교육자료
		ㄴ 사용자 계정(관리자 계정, DB 계정, API 키, 인증 토큰)

	2. 개발 환경 서버 조사
		ㄴ 어떤 환경이 내 서비스에 맞을지 분석
		ㄴ 서버의 종류, 운영체제, 성능, 보안, 비용 등을 분석하는 과정

	3. 운영체제별 특징 파악
		

윈도우 명령어
	파일/폴더 관리 명렁어
		mkdir 폴더명 : 폴더의 생성
		cd 경로 : 폴더의 이동

	파일 목록 확인
		dir : 현재 폴더의 파일 목록을 출력

	파일 삭제
		del 파일명 

	폴더 삭제 
		rmdir 폴더명 /s /q
		/s : 내부 파일까지 삭제
		/q : 확인 없이 삭제

	ip 주소 확인
		ipconfig

	네트워크 연결확인
		ping 주소 : 인터넷 연결 테스트
	
	실행중인 프로세스 확인
		tasklist

	현재 사용자 확인
		whoami

	입력한 내용을 그대로 출력하는 명령어
		echo 내용 > 파일명

		> : 덮어쓰기
		>> : 추가 쓰기

	type
		파일에 들어있는 내용을 터미널에 출력하는 기능
		간단한 텍스트 내용 확인
		
		type 파일명1, 파일명2 : 여러개의 파일을 이어서 출력할 수 있다
		type 파일명1 > 파일명2 : 파일명 1의 내용을 파일명2로 저장
		type 파일명1, 파일명2 > 두 타입을 하나로 합친다

	환경변수
		시스템이나 프로그램이 사용하는 "설정 값"
		
		기본적인 사용법
		%변수명%
		%cd% : 현재 작업중인 폴더 값이 들어있는 변수
		%username% : 현재 로그인 사용자
		%windir% : 윈도우 설치 경로
		%random% : 0 ~ 32767 사이의 숫자 출력

	
wsl(Windows SubSystem for Linux)

	윈도우 안에서 리눅스 환경을 실해애할 수 있게 해주는 기능
	윈도우를 그대로 쓰면서도 bash, ls, pwd, apt, git과 같은 
	리눅스 명령어와 도구를 사용할 수 있다
	전통적인 가상머신이나 듀얼부팅 없이 windows에서 
	GNN/Linux 환경을 직접 실행할 수 있다
	
	가상머신보다 가볍고 편하기 때문에 사용을 한다

	wsl을 통해 리눅스를 설치하게 되면 Linux 커널을 기반으로
	만들어진 Ubuntu가 설치가 된다

sudo( Superuser Do)
	리눅스에서 관리자 권한으로 실행해라 라는 뜻

	우분투에서는 일반 사용자가 아무 프로그램이나 시스템에 설치하거나 
	바꾸지 못하게 막아둔다
		ㄴ 실수로 시스템이 망가지는 것을 방지
		ㄴ 보안 강화
		ㄴ 중요한 파일 보호

	시스템 전체에 영향을 주는 명령어는 보통 sudo가 필요하다

apt
	apt 우분투에서 프로그램을 관리하는 도구
	정확히는 패캐지 관리자라고 부름
		ㄴ 프로그램
		ㄴ 라이브러리
		ㄴ 설치파일의 묶음
		ㄴ 업데이트 파일

	윈도우에서는 인터넷에 접속하여 .exe 파일을 직접 다운로드해서 설치한다
	우분투는 apt를 통해 설치한다


sudo apt update : apt가 가서 다운받는 저장소의 목록을 갱신
	ㄴ 설치 목록이 오래됐을 수 있기 때문에
	ㄴ 설치 오류가 날 수도 있다
	ㄴ 구 버전을 설치하게 될 수도 있다

	저장소에서 정보를 받아와야 하므로 인터넷이 연결되있어야 한다
	시스템 패키지 정보에 영향을 주기 때문에 sudo를 붙여야 한다
