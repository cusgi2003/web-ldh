Map 인터페이스

	Key와 Value가 한 쌍으로 하나의 데이터로 저장

HashMap 

	Map 인터페이스를 구현한 대표적인 클래스

	put(key, value) : 데이터의 삽입
		ㄴ 있으면 추가, 없으면 value의 갱신이 발생
	get(key) : 해당 키와 연결된 value값 반환
	getOrDefault(key, default) : 해당 key의 value가 있으면 반환, 없으면 default값을 세팅
	remove(key) : map에 들어있는있는 데이터 삭제
	size() : 컬렉션의 크기를 반환
	 isEmpty() : 컬렉션이 비어있는지 검사 
	containskey(key) 	: 해당 ket가 포함되어있는지 검사
	containinns(key  	: 해당valye가 포함되어있는지 검사

	keySet() 	: 메서드에 들어있는 모든 key만 추출하여 Set에만 저장
	values() 	: map에 들어있는 모든 value들만 가져오기 
	entrySet() 	: map에 들어이있데 데이터터를 하나씩 꺼내서 Set에 저장


객체지향프로그래밍에서의 메서드 사용법
	
	1. 클래스 생성
	2. 클래스 안에 메섣를 작성
	3. 그 클래스의 객체를 생성
	4. 객체로부터 메서드 호출


함수형 프로그래밍

	함수를 정의하면 객체 생성없이 호출하는게 가능하다
	다른 함수로 정의했다면 import만 하면 사용할 수 있다


람다식

	1. 간단하게 작성할 서수 있다
	2. 핵심로직만 쓴다

	람다식을 구현하려면 함수형 인터페이스가 필요하다
		추상 메서드가 딱 1개만 있는 인터페이스

	람다식 기본 문법
	
		반환타입과 메서드명은 제거
		(매개변수) -> {실행문}

	매개변수가 1개일 때
		소괄호는 생략이 가능하다
		매개변수의 타입 생략이 가능하다

	매개변수가 2개일 때
		타입을 생략하려면 전부 다 생략해야 한다
		소괄후 생략은 불가능하다

	반환할 값이 없고 명령이 1줄이면 중괄호 생략이 가능하다
	명령이 1줄이고 return과 중괄호를 같이 생략할 수 있다
	함수식을 변수에 담을 수 있다
	변수를 통해서 함수를 호출한다

	매개변수에 함수를 전달한다
	반환값으로 함수를 반환한다
