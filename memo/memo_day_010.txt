상속

	관련있는 것들끼리만 묶는 것
	공통적인 속성(필드, 멤버)나 동작(메서드)을 부모쪽에 몰아놓는 것
	자식은 부모클래스를 상속받아서 사용

	N차 상속이 가능함 
		상속 -> 상속 -> 상속
	꼭 부모가 아닌 조상의 필드나 메서드를 사용할 수 있다
		Car -> Bus -> SchoolBus
	
	다중상속이 불가능하다

	* Object : 모든 클래스의 부모 클래스

	seald : 봉인된 클래스
	permit : 내가 허용한 클래스만 상속을 할 수 있게 한다


인터페이스 
	
	동작에 대한 가이드라인 제공
	같은 범주에 있는 것들이 아니여도 됨
	
	추상클래스와 마찬가지로 직접 객체를 만드는 것이 불가능하다.
	인터페이스를 구현할 클래스가 있어야 한다

	class Impl implements 인터페이스명{};

	추상메서드를 반드시 오버라이딩해야 한다
	다중구현이 가능하다
	각 인터페이스에 있던 추상메서드를 전부 오버라이딩 해야한다


중첩클래스
	많이 사용하진 않지만 특정한 상황에서 쓰인다
		1. 클래스 내부에서만 쓰는 전용 객체	


정적내부 클래스
	내부클래스인데 static이 붙은 형태이다
	바깥 객체에 소속된 내부 클래스처럼보이지만, 실제로는 바깥 객체 없이도
	독립적으로 만들 수 있는 내부 클래스
