운영체제

	하드웨어와 소프트웨어 중간을 담당하는 프로그램
		Windows, macOS, IOSS, Android, Linux, Unix

	개발하려는 목적에 따라 운영체제를 선택을 잘 해야 한다
		웹 개발 : Windows, 서버운영 : Linux, Unix
		웹 갭발 : MacOS(애플 전용앱)
		앱 실행 : IOS, Android

	운영체제가 하는 일
		주어진 시간 내에 많은 일을 처리하기
		사용자 요청에 빠른 응답하기
		오류 없이 안정적으로 동작하기
		사용자에게 인터페이스 제공하기


	- 한 컴퓨터에서 인터넷 브라우저, 문서 편집기, 메신저가 동시에 실행되고 있다
	- 이 상황에 관련된 운영체제의 기능을 2가지 쓰고, 이유를 설명


	cpu 실행관리;
		여러 프로그램이 cpu를 나눠서 사용하도록 조정해야하기 때문이다
	주기억장치
		여러 프로그램이 동시에 메모미를 사용하므로 메모리를 적절히 할당해야한다

쉘(shell)

	사용자가 입력한 명령어를 운영체제에 전달해서 실행해주는 "명령어 해석기"

	우리가 명령어를 입력하는 공간 -> 터미널
	Shell은 터미널 안에서 동작하는 프로그램


	윈도우에서도 Shell이 있다
		1. Command Prompt (cmd)
			ㄴ 가장 오래된 Windows 기본 Shell
		2. PowerShell
			ㄴ 현재 Windows의 기본 Shell
			ㄴ 객체 기반 명령어 사용
			ㄴ 자동화, 스크립트 매우 강력하다
		
	현재 터미널 세션에서만 사용 가능 -> 터미널을 닫으면 변수가 사라진다


echo는 파일 작업에서 가장 많이 사용된다

	1. 파일 생성 및 덮어쓰기
		echo "Hello" > file.txt
		ㄴ file.txt 생성
		ㄴ 기본 내용 있으면 덮어쓴다
	2. 파일에 추가
		echo "World" >> file.txt
		ㄴ 기존 내용 유지
		ㄴ 뒤에 추가됨

	
	nice : 실행하려는 프로세스의 우선순위를 설정
	nice -n 우선순위 명령어
		ㄴ 일반 사용자는 순위를 낮추는 것만 가능하다	
		ㄴ 순위를 높이려면 관리자 권한이 필요

프로그램을 백그라운드에서 실행하기
	ㄴ 명령어를 실행하면서도 터미널을 게속 사용할 수 있게 하는 방법

	포어그라운드(Foreground) -> 현재 터미널을 점유
	백그라운드(Background) -> 뒤에서 실행 (터미널 사용가능)

	기본적인 방법은 명령어 뒤에 & 붙이기
	Ex) sleep 100 &
	
	[1] 12345
	[1] -> 작업 번호
	12345 -> PID

	파일 소유권
		어떤 사용자와 어던 그룹이 그 파일의 주인인가를 나타내는 정보

	리눅스에서 파일 하나에는 2가지의 소유 정보가 붙는다
		소유 사용자(User Owner)
		ㄴ 해당 파일의 대표 사용자
		소유 그룹(Group Owner)
		ㄴ 해당 파일과 관련된 대표 그룹
		ㄴ 같은 그룹에 속한 사용자들은 그룹 범위 안에서 
		    접근할 수 있다

	왜 파일 소유권이 필요한가
		보안과 역할 분리 때문이다

파일 허가권
"누가 이 파일에 대해, 무엇을 할 수 있는가"를 정하는 규칙

컴퓨터를 여러 사용자가 함께 쓰거나, 서버에서 여러 프로그램이 동시에 동작하면
아무나 파일을 읽고, 수정하고, 실행할 수 있으면 큰 문제가 생긴다

무엇을 할 수 있는가
권한의 종류는 3가지
r : read
w : wriite
x : execute

파일에서의 r,w,x의 의미
r (read)
ㄴ 파일의 내용을 읽을 수 있다
ㄴ 텍스트 파일 열기
ㄴ cat을 이용하여 내용 보기
ㄴ 프로그램이 파일 내용 읽게 하기

w(write)
ㄴ 파일의 내용을 수정할 수 있다
	ㄴ 내용 추가
	ㄴ 내용 삭제
	ㄴ 내용 덮어쓰기

x(execute)
ㄴ 파일을 실행할 수 있다
ㄴ 쉘 스크립트 실행
ㄴ 실행파일 실행

디렉터리에서 r,w,x의 의미
r : 디렉터리 안에 어떤 파일이 있는지 목록을 볼 수 있다
w : 디렉터리 안에서 파일 생성, 삭제, 이름 변경
x : 디렉터리 안으로 들어갈 수 있는 권한 

- rw- r-- r--
첫글자 : 파일의 종류
- : 일반파일
d : 디렉터리
l : 링크

rw- : 소유자는 읽기, 쓰기만 가능
r-- : 그룹은 읽기만 가능
r-- : 다른 사용자는 읽기만 가능


권한을 숫자로도 표현할 수 있다

r =4
w=2
x=1
이 값을 더해서 표현한다
rwx => 4 + 2 + 1 = 7
rw- => 4 + 2 = 6
r-x => 4+1 = 5


chmod 

chmod 644 memo.txt
소유자 : rw- 
그룹 : r--
기타 : r--


문자방식
u : user
g : group
o : other
a : all
연산자 
+ : 권한추가
- : 권한제거
= : 권한을 지정한 값으로 설정

chmod u+x test.txt
소유자에게 실행권한을 추가

chmod g-w memo.txt
그룹의 쓰기 권한을 제거

chmod o-r memo.txt
기타 사용자의 읽기 권한을 제거

파일 소유자가 아니면 권한을 바꿀 수 없다

파일소유자 변경하기
chown(change owner)

sudo chonwn 새소유자 파일명

일반 사용자는 자기 마음대로 소유자를 바꾸지 못하고
관리자 권한이 필요하다

소유 그룹을 변경하는 명령어
chown 사용자:그룹 파일명

디렉터리에서 하위 파일까지 한 번에 소유권 변경하기
디렉터리 안에서 파일이 많을 때는 -R 옵션을 사용한다
sudo chown -R 사용자:그룹명 디렉터리명
