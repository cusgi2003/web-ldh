참조용  클래스(객체를 만들기 위한 클래스)

	main 메서드가 없음

	필드

	생성자(호출했을 때 메모리에 객체를 올려주는 기능)
		객체를 생성했을 때 필드르 초기화하는 역할

	메서드(여러개 정의가 가능하다)


접근제한자 반환타입 메서드명(매개변수) {
}


메서드
	ㄴ 정의(기능을 만드는것)
		ㄴ 숫자 두개와 연산자를 받을 준비

	ㄴ 호출
		ㄴ 숫자와 연산자를 전달하여 연산결과를 돌려받는다


return

	매개변수, 메서드 내부에서 정의된 변수는 지역변수이다
	메서드가 호출되서 실행한 다음 종료가 되면 메모리에서 삭제된다

	만약, 메서드 밖에서 연산결과를 사용하고싶다면
	return을 통해 값을 밖으로 꺼내야 한다


메서드명

	메서드를 호출할 때 사용하는 이름
	첫 글자는 소문자로 작성


생성자

	객체를 생성했을 때 필드를 초기화하는 역할
	사용자가 직접 정의하면 컴파일러의 눈에만 보일 뿐, 우리가 보는 코드에는 생략되어있다
	
	매개변수와 클래스의 맴버변수의 이름이 같다면
	매개변수의 사용 우선순위가 더 높다
	생성자를 호출한 객체의 필드임을 명시하기 위해서 this 키워드를 사용한다

	static 키워드가 붙어있으면 메모리에 한번만 올라간다
	
	
오버로딩(overloading)

	같은 이름으로 서로 다른 개수, 타입의 매개변수를 각각 처리하는 방법
	
	반환형은 달라도 상관없음
	메서드(or생성자)명은 같아야 한다
	매개변수의 개수가 다르거나, 타입이 다르거나, 순서가 달라야한다


접근제한자(Access Modifier)

	public  : 프로젝트 내에서 어디서든지 사용 가능
	default : 같은 패키지 내의 다른 클래스 어디서든지 사용 가능
	private : 현재 클래스에서만 사용 가능
	protected
	
	필드 / 생성자 / 메서드 / 클래스에 붙힐 수 있다.
	


클래스 만들기

	필드
		문자열 타입의 이름
		정수 타입의 이름

	모든 필드는 private로 선언
	getter / setter를 각각 구현



User클래스 작성

	필드
		문자열 username
		문자열 password
		문자열 email

setter&getter를 이용해서 필드의 값을 세팅하거나 가져올 수 있다


디자인 패턴

	프로그래밍을 하다보면 비슷한 문제가 반복적으로 나타나는 경우가 있다
		ㄴ객체를 하나만 만들어야하는 경우
		ㄴ객체 생성이 너무 복잡한 경우
		ㄴ기능은 비슷한데 구현 방법이 여러개인 경우
		ㄴ객체들끼리 너무 강하게 연결되어 있어 수정이 어려운 경우

	개발자들이 오랜 시간 동안 많이 쓰인 설계 방법을 정리해둔 것
