MySQL
관계형데이터베이스
- 테이블 형태로 데이터를 관리
- 행 : 가로 데이터셋(한 건의 데이터)
ㄴ 행의 개수 : 카디널리티
- 열 : 세로 데이터셋(한 가지의 데이터)


SQL의 종류
ㄴ DDL : 데이터베이스 객체를 조작
	ㄴ CREATE
	ㄴ ALTER
	ㄴ DROP
ㄴ DML  : 테이블 내의 데이터를 조작하는 SQL
	ㄴ SELECT
	ㄴ INSERT
	ㄴ UPDATE
	ㄴ DELETE
ㄴ DCL : 권환 관리 명령어
	ㄴ GRANT
	ㄴ REVOKE
ㄴ TCL : 트랜잭션 제어
	ㄴ COMMIT
	ㄴ ROLLBACK

트랜잭션 : 여러 SQL 작업을 하나의 작업 단위를 묶는 것

MySQL은 기본적으로 AUTO COMMIT 상태로 되어있다
START TRANSACTION
쿼리문1

ROLLBACK::

제약조건
- 테이블에 잘못된 데이터가 들어가지 않도록 막는 규칙


모델링
- 현실세계의 데이터를 데이터베이스 구조로 설계하는 과정
요구사항 분석
개념모델링 - 엔티티 결정(ERD작성)
논리모델링 - ERD를 보고 자료형/제약조건 설정, 정규화
물리모델링 - 실제 테이블로 만드는 과정

SQL 함수
단일행 함수 - 1개의 행 값이 함수에 적용되어 1개의 행을 반환하는 것
집계함수 - 1개 이상의 행 값이 함수에 적용되어 1개의 행을 반환한다

문자함수
LENGTH() : 바이트 수 반환
CHAR_LENGTH() : 글자 수 반환
UPPER(), LOWER() : 대문자/소문자 변환
TRIM(), LTRIM(), RTRIM() : 양쪽공백/왼쪽공백/오른쪽공백 제거
LPAD(), RPAD() : 문자열 자지수 지정, 남으면 특정 문자열로 채움
SUBSTRING(), SUBSTR(), MID() : 문자열 자르기
※ MySQL 인덱스는 1부터 시작한다

LEFT(), RIGTH() : 문자열을 왼쪽/오른쪽부터 내가 원하는 만큼 자른다
INSTR() : 문자열의 위치번호 찾기
CONCAT() : 나열된 문자열의 연결
REPLACE() : 문자열에서 특정 문자열을 다른 문자열로 교체 
(등장하는 모든 문자열을 교체해준다)
	ㄴ 없으면 원래 문자열 그대로 반환한다
