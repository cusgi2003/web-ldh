객체지향 프로그래밍(object-oriented Programing)

	프로그램을 객체 단위로 나눠 설계하고 구현하는 방식


객체지향을 사용하지 않을 때 문제점

	데이터와 기능이 따로논다
	코드가 많아지면 관리가 어렵다
	재사용이 힘들다


객체지향의 핵심개념

1. 클래스
	설계도, 우리가 만든 참조자료형
	Ex. 사람을 어떻게 정의할건지, 제품을 어떻게 정의할건지

	정의할 수 있는 내용
		필드 : 각 객체가 가지는 정보(상태, 속성)
		메서드 : 각 객체가 할 수 있는 동작

2. 객체
	클래스명 변수명 = new 클래스명();
	클래스를 통해서 만들어진것

	필드에 객체만의 정보를 넣거나 필드의 내용을 사용할 수 있다
	객체변수명.필드명

	클래스에 정의된 기능을 호출하여 사용할 수 있다
	객체변수명.메서드명()


객체지향의 4대 특징
	캡슐화
		데이터를 보호하는 개념
		필드 직접 접근하지 못하게 한다
		제공된 메서드를 통해서만 접근할 수 있게 한다.

	상속
		기존 클래스의 내용을 몰려받는 기능(필드,메서드)
		코드를 재사용할 때 좋음

	다형성
		하나의 타입으로 여러객체를 다룰 수 있는 성질
		타이어 마다 성능이 다르다

	추상화
		클래스를 단순화 시키는 것
