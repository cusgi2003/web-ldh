Stream API

	배열, 컬렉션 등에 들어있는 데이터를 가공, 필터링, 변환, 집계하는
	등 작업을 더 선언적으로 작성할 수 있게 해주는 기능
	
	※ 선언적

		"무엇을 할것인지"를 중심으로 코드를 작성하는 방식
		어떻게 처리할지(과정)은 신경쓰지 않고 결과로
		무엇을 얻고싶은지(목표)만 표현하는 것

	선언형(결과중심)
	
		메서드를 호출해서 결과만 확인하는 방식
		
	핵심 특징

		1. 원본 데이터를 직접 바꾸지 않는 경우가 많다
		2. 데이터 흐름을 중심으로 작성한다
		3. Stream객체는 일회용이다
			ㄴ 한 번 사용한 객체는 다시 사용할 수 없다
		4. 지역연산이 가능하다
			ㄴ 결과를 바로 주는게 아니라 나중에 돌려주는 작업

Stream API를 사용하는 방법

	1. 스트림 객체 생성
		컬렉션이나 배열을 통해 스트림을 생성
	2. 스트림의 데이터를 가공한다
		개수를 필터링하거나, 변환하거나, 중복을 제거 등등
		가공은 여러개를 이어서 사용할 수 있다
	3. 최종연산
		가공된 결과를 다 더하거나, 개수를 세거나, 출력하거나
		가공된 결과를 돌려준다

	컬렉션.stream()
		.중간연산()
		.중간연산()
		.중간연산()
		.최종연산();
