[Q1] UI 설계서의 첫 페이지에 위치하며, 여러 프로젝트 문서 중 해당 문서의 정체성을 구분하기 위해 프로젝트명 또는 시스템명을 반드시 명시해야 하는 구성 요소는 무엇인가?
>UI 설계서 표지

[Q2] UI 설계서가 사용자 의견이나 기획 변경, 개발 검토 결과에 따라 수정된 기록을 관리하는 부분으로, '누가, 언제, 무엇을 변경했는지'와 '현재 최신 버전'을 확인하기 위해 작성하는 영역은 무엇인가?
> UI 설계서 개정 이력

[Q3] UI 요구사항 중 '로그인, 검색, 저장'과 같이 사용자가 시스템을 통해 직접 수행할 수 있어야 하는 행위 중심의 요구사항 유형을 무엇이라고 하는가?
> 기능 요구사항

[Q4] UI 요구사항 중 '버튼의 크기, 색상, 응답 속도'와 같이 시스템이 제공하는 기능의 품질이나 제약 조건을 규정하는 요구사항 유형은 무엇인가?
> 비기능 요구사항

[Q5] 사용자와 컴퓨터의 상호작용(HCI)에 대한 국제 표준 중, 화면 설계 시 자주 사용되는 기본 UI 구성 요소를 버튼, 체크 박스 등을 포함하여 총 51개 항목으로 제시하고 있는 표준 번호는 무엇인가?
> ISO 9241-161

[Q6] 여러 개의 UI 구성 요소를 담는 영역으로, 카드 뉴스나 대시보드의 개별 컴포넌트처럼 화면 안의 공간을 구분하고 제목과 콘텐츠 영역을 포함하여 사용자에게 정보 구조를 제공하는 UI 요소는 무엇인가?
> 컨테이너

[Q7] 사용자가 시스템을 활용하여 자신이 원하는 목표를 얼마나 효과적이고 효율적으로 달성할 수 있는지를 나타내는 지표로, 쉽게 말해 '사용하기 쉬운 정도'를 뜻하는 UI 설계의 핵심 개념은 무엇인가?
> 유용성

[Q8] 서비스 구축 과정에서 디자이너와 개발자가 공통으로 참고하는 최종 산출물로, 와이어프레임, 기능 정의, 유효성 검사, 예외 처리, 데이터베이스 연동 정보 등을 모두 포함하는 문서는 무엇인가?
> 스토리보드

[Q9] UI 설계서에서 화면의 구체적인 이름이 있음에도 불구하고 UI-S001, UI-S002와 같은 고유한 화면 ID를 부여하여 관리해야 하는 이유를 실무적 관점(의사소통 및 개발 범위 관리)에서 서술하시오.

[Q10] 쇼핑몰 앱에서 사용자가 상품을 구매하려는 목적을 가질 때, '프로세스 정의(Process Definition)'가 명확하지 않으면 사용자에게 어떤 불편함이나 문제(최소 2가지 이상)가 발생하는지 구체적인 예시를 들어 서술하시오.

[Q11] UI 구성 요소 중 '체크 박스(Check Box)'와 '라디오 버튼(Radio Button)'의 기능적 차이점을 '선택 가능한 항목의 개수' 관점에서 비교하여 서술하시오.

[Q12] '평가 차(Gulf of Evaluation)'의 정의를 기술하고, 시스템이 파일 업로드나 도서 예약을 처리할 때 평가 차를 최소화하기 위해 제공해야 하는 피드백(Feedback) 방법을 기술하시오.
