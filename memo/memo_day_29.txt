정규화 논리모델링을 하면서 진행
정규화
- 데이터 중복을 줄이고, 데이터 이상현사을 방지하기 위해 테이블을
구조적으로 나눈 것

===========================

함수적 종속성
- 어떤 속성의 값이 정해지면, 다른 속성의 값도 자동으로 결정되는 관계
A -> B

완전함수 종속
- 기본키 전체에 의해 종속된느 경우
학번		과목코드	성적
1		DB		A
1		JAVA		B

부분함수 종속
- 기본키의 일부만으로 종속되는 경우
학번		과목코드	학생이름	과목명
1		DB		김철수	데이터베이스

이행함수 종속
A -> B, B -> C면 A -> C가 되는 관계
학번		학과코드	학과명
1		D01		컴퓨터공학
학번 -> 학과코드, 학과코드 -> 학과명 일 때 학번 -> 학과명
학과명이 중복저장됨
수정 이상이 발생한다

이상현상이란?
데이터베이스 설계가 잘못되어 데이터를 삽입, 수정, 삭제할 때 
비정상적인 문제가 발생하는 현상

삽입 이상
- 데이터를 추가혀록 할 때, 필요 없는 데이터까지 강제로 넣어야 하는 문제

학생번호	학생명	과목명	교수명
1		김철수	DB		이교수
null		null		PYTHON	김교수
-과목만 추가하고싶은데 학생 정보까지 필요

수정 이상
- 같은 데이터가 여러 번 저장되어 일부만 수정하면 
데이터의 일관성이 사라지는 문제
학생번호	학생명	과목명	교수명	강의실
1		김철수	DB		이교수	301
2		이영희	DB		이교수	301
3		박민수	DB		이교수	301

삭제 이상
- 데이터를 삭제할 때, 유지해야할 다른 정보까지 같이 삭제되는 문제
