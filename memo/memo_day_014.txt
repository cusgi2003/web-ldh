Class 클래스의 역할

	클래스의 정보를 알아와야 할 때
	생성자를 읽어서 객체를 만드는 것
	ㄴ 프로그램이 실행되는 시점

	핵심
		컴파일 시점에 확인하는 것이 아니라 실행시점에서
		확인하기 위함이다


어노테이션

	코드에 붙이는 표식같은 것
	클래스, 필드, 메서드 등에 붙여서 코드에 대한 
	추가 정보를 전달할 때 사용

	코드 자체의 실행문은 아니지만 이 코드가 어떤 의미인지,
	컴파일러가 무엇을 확인해야 하는지, 프레임워크가 어떻게
	처리해야 하는지와 같은 정보를 알려주는 역할

	1. 실수를 줄여준다
	2. 코드의 의미를 더 분명하게 보여준다

	자바가 제공하는 어노테이션
		@Override
		@Dreprecated : 더이상 업데이트 안되니 그만 써 
		@SupressWarnings : 경고 보기 싫을 때

	메타 어노테이션
		@Retention : 언제까지 정보를 유지할거냐
		@Target : 어디에 적용할거냐

	나만의 어노테이션
		public @Interface 어노테이션명{}
