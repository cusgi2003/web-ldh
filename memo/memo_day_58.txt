문단
<p> ~ </p> : 문단을 나눌 때 쓰는 태그 안에 내용을 작성

미디어 태그
<img src="경로"> : 웹 페이지에서 이미지를 표시하기 위해 사용

절대경로
특정 문서 페이지 또는 이미지 등 자원에 접근할 수 있는 전체 URL

상대경로
현재 열려있는 파일로부터 목적지 파일까지의 경로
./ : 현재 파일 위치
../ : 현재 파일의 한 단계 상위 경로
../../ : 현재 파일의 두 단계 상위 경로

시맨틱 태그
- 웹 페이지 내에서 콘텐츠가 어떤 역할을 하는지 브라우저와 사람에게 알려주는 태그
- 콘텐츠를 논리적 섹션으로구성하고 각 부분의 역할과 기능을 전다랗느 ㄴ데 도음이 된다

검색엔진에 최적화
- 구글과 같은 검색엔진 상에서 웹사이트 순위에 영향을 미친다.
- 시맨틱 태그를 사용하면 검색엔진과 사용자에게 웨베페이지가 더 눈에 띄고 매력적으로 보이게 만들 수 있다.

접근성 향상
- 스크린리더 등 보조기기가 콘텐츠를 올바르게 해석한다

유지보수 용이성
- 코드 구조가 명확해져 협업과 수정이 쉬워진다

가독성 향상
- 의미가 드러나는 태그 이름으로 코드 이해가 쉬워진다.

시맨틱 태그의 종류와 역할
<header> : 사이트의 도입부, 로고, 제목, 검색창, 내비게이션
<nav> : 내비게시연 표현을 위한 태그로서 동일한 사이트 안의 문서나 다른 사이트이
문서로 연결하는 링크들의 모음

<main> : 해당 페이지의 메인 콘텐츠를 나태낼 때 사용
<section> : 주제별로 묶이는 콘텐츼 덩어리를 나타내는 요소
<article> : 독립적으로 재사용 가능한 글이나 게시물 단위
<aside> : 보조 정보(광고, 참고, 링크 등)
<footer> : 페이지나 섹션의 바닥글(저작권, 연락처, 소셜 미디어 링크)

스타일을 적용하는 방법
1. 인라인
2. 내부 스타일시트
3. 외부스 타일시트
3. @import

스타일을 적용하기 위한 요소를 선택하는 방법
1. 태그선택
2. 클래스 선택
3. 아이디 선택
4. 속성 선택
5. 하위 선택(공백)
6. 직계 자식 선택(>)
7. 인접 형제 선택 (+)
8. 일반 형제 결합 (~)
