요구사항 확인

1. 현행시스템의 분석
새 시스템을 만들거나, 기존 시스템을 개선하기 위해 현재 사용중인 업무 방식, 프로그램, 데이터,
서버, 네트워크, 문제점을 조사하고 정리하는 활동

목적
- 새로운 시스템을 정확하게 만들기 위해 현재 상황을 이해한다

문제점
1. 기존 업무의 흐름을 모른다면 필요한 기능을 빠트릴 수 있다
2. 기존 데이터 구조를 모른다면 데이터 이전 시 오류 발생
3. 기존 시스템 연동을 모른다면 외부 시스템과 연결 실패
4. 사용자 불편 사항을 모른다면 새 시스템을 만들어도 만족도가 낮다.
5. 현재 원인을 모른다면 문제 해결이 아닌 단순 재개발이 된다.


현행 시스템 파악 절차

1단계 : 현행 시스템의 구성, 기능, 인터페이스 현황 파악
- 현재 시스템이 무엇으로 구성되어 있고, 어떤 기능을 제공하고,
다른 시스템과 어떻게 연결되어 있는지 파악

시스템 구성현황 파악
- 현재 운영중인 시스템의 종류와 역할을 정리하는 것
- 각 시스템이 어떤 업무를 담당하는지까지 정리한다

기능 현황 파악
- 각 시스템이 제공하는 기능을 정리하는 것

수강생 관리 시스템
수강생 등록 : 신규 수강생을 등록
수강생 조회 : 학원에 다니는 수강생을 조회
수강생 수정 : 수강생의 정보를 수정
수강생 삭제 : 잘못 등록된 수강생 정보를 삭제

인터페이스 현황 파악
- 시스템과 시스템이 데이터를 주고받는 연결 지점을 의미
- ex 쇼핑몰
결제 api -> 주문번호, 결제금액, 결제상태
택배사 api -> 주문번호, 주소, 송장번호
문자 발송 api -> 전화번호, 메세지 내용
이메일 api -> 이메일 주소, 인증 코드
cupang.com/search?q=데이터


2단계 : 현행 시스템의 아키텍쳐 및 소프트웨어 구성 현황 파악
- 현재시스템이 어떤 구조로 만들어져 있고, 어떤 소프트웨어를 사용하고 있는지 파악

아키택처 현황 파악
- 현재 시스템이 어떤 구조로 만들어져 있는지 조사하는 일

아키텍처
- 시스템의 구성

소프트웨어 구성 현황 파악
- 현재 시스템이 어떤 소프트웨어를 사용하고 있는지 조사하는 일

2단계 산출물
- 시스템 아키텍처 구성도
- 클라이언트부터 인프라까지 전체 구성을 한 눈에 파악할 수 있다
시스템을 구성하는 소프트웨어를 영역별로 정리한 목록
- 소프트웨어 버전 현황표
현재 사용버전, 최신 버전, 업데이트 상태를 한 눈에 파악할 수 있는 표


3단계 : 하드웨어 및 네트워크 분석
- 현재 시스템이 운영되는 물리적 또는 클라우드 인프라 환경을 파악한다.

서버는 어디에서 운영되는지?
서버 사양은 어떻게 되는지
네트워크는 어떻게 연결되어 있는지
방화벽이나 보안 설정은 어떻게 되어있는지

포트(port)
- 외부와 데이터를 주고받는 통로이자 접속 지점
SSL(Secure Sockets Layer)인증서
- 웹사이트와 사용자 브라우저 간 데이터를 암호화하여 개인정보 노출을 막는
디지털 보안 솔루션
- 데이터 암호화
데이터가 전송 중 변조되지 않도록 보호
-신원 보호
클라이언트가 접속하는 서버가 신뢰할 수 있는 서버인지 확인

캐시(Cache)
- 자주 사용하는 데이터나 값을 미리 복사해 놓는 임시 장소를 가리킨다.
- 저장공간이 작고 비용이 비싼 대신 빠른 성능을 제공한다.

- 발열이 심하고, 너무 비싸져서 현실성이 없다

클러스터(Cluster)
- 여러 대의 독립된 컴퓨터나 서버, 혹은 데이터들을 하나로 묶어 '단일 시스템'
처럼 작동하게 만드는 기능 및 기술

SSMS(Sql Server Managerment Studio)
- Microsoft SQL Server를 관리하기 위한 전용 프로그램

Active Directory
- windows 계정과 사용자 권한을 중앙에서 관리하는 시스템

.NET
- Microsoft의 개발 플랫폼
- C#이 포함된다.

튜닝
- 데이터베이스가 더 빠르고 효율적으로 동작하도록 최적화하는 작업
- 조회 속도 개선
- cpu 사용량을 감소
- 메모리 효율 증가
- 대량 동시접속 처리
- 디스크 I/O를 감소

실행계획 최적화 기능이 상대적으로 약함
DBMS는 SQL 실행 전에 "어떤 방식으로 데이터를 찾는게 가장 빠를까?" 를 계산
이걸 실행 계획이라고 한다
MySQL도 실행 계획이 있지만, Oracle과 같은 상용 DB보다는 세밀함이 부족한 경우가 있다

옵티마이저
- SQL 실행 전략을 자동으로 결정하는 엔진
- 오라클에 비해서는 세밀하지 못하다

대규모 병렬처리에 약함
MySQL은 전통적으로 단순 웹서비스 중심, 읽기 위지의 구조에 강점이 있다
그래서 초대형 데이터 분석에는 한계가 있었다.


Redis : 데이터를 메모리에 저장함
실시간 메세지 기능 
pulish -> 메시지 보내기
subscribe -> 메시지 받기

누군가는 메세지를 발행하고, 누군가는 그 메시지를 구독해서 실시간으로 받는다.

채팅
실시간 알림
주식 시세
게임 상태 동기화

1. 운영체제 역할 설명하기
- 하드웨어, 소프트웨어 자원을 관리한다
- CPU, 메모리, 디스크 등을 프로그램에 할당한다.
	없는 경우 발생하는 문제
		프로그램 실행이 어렵다
		CPU와 메모리를 직접 제어해야 한다
		여러 프로그램을 동시에 실행하기 어렵다


2. 운영체제의 종류 구분하기
WINDOWS
	개인용, 중소규모 서버에 많이 사용된다
	GUI를 사용한다
LINUX
	오픈소스이다(무료다)
	서버환경에서 많이 사용한다
UNIX
	안정성이 높고, 대규모 서버 환경에 사용한다


3. 신뢰도 요구사항 분석하기
- 운영체제 선정 시 신뢰도가 중요한 이유를 작성하시오.

	운영체제 장애 발생 가능성을 줄여야 한다
	메모리 누수 발생 시 성능 저하가 발생할 수 있다
	보안 패치 및 버그 수정시 안정적인 운영이 필요하다
	서버 중단 시 결제 오류와 서비스 장애가 발생할 수 있다.


4. 메모리 누수 설명하기
- 개념과 문제점에 대해 설명하기
	개념 : 프로그램이 사용한 메모미를 반납하지 않는 현상
	문제점
		메모리 사용량이 계속 증가한다
		서버 속도가 느려진다
		프로그램 오류가 발생할 수 있다
		결국 서버를 재기동할 필요가 있다


5. 동시 사용자 처리 분석하기
	CPU 사용량 분배
	메모리 할당
	네트워크 연결 관리
	프로세스 스케줄링
	사용자 요청 처리 순서 결정

	
6. 대용량파일 처리 요구사항 분석하기
영상 업로드 서비스에서 운영체제의 파일 처리 성능이 중요한 이유
	대용량 파일 저장 작업이 많기 때문이다
	디스크 I/O 성능이 중요하다
	여러 사용자가 동시에 업로드할 수 있다
	파일 저장 속도가 느리면 서비스 성능이 저하된다
	서버 과부하가 발생할 수 있다


7. 32bib / 64bit 운영체제 차이점 설명하기
	32bit
		메모리 사용 약 4GB 제한
		성능이 제한적
		일반 PC
	64bit
		메모리 4gb 이상 사용 가능
		대규모 작업 처리 가능
		서버 및 고성능 시스템


8. 운영체제 선성시 공급 배너의 기술 지원이 중요한 이유
	장애 발생 시 빠른 대응이 가능하다
	보안 패치를 안정적으로 받을 수 있다
	공식 문서와 메뉴얼을 제공받을 수 있다
	유지 보수가 쉬워진다
	장기 지원 버전을 사용할 수 있다


9. 사용자가 많은 운영체제를 사용하는 장점을 설명하세요
	검색 자료가 많다
	문제 해결 방법을 쉽게 찾을 수 있다
	커뮤니티 활동이 활발하다
	에제 코드와 기술 문서가 풍부하다
	유지보수 인력을 구하기 쉽다.
