# 요구사항


## UI(User Interface) : 사용자 인터페이스
- 사용자가 제품과 상호작용하는 시각적인 요소와 디자인을 의미한다.  
- UI는 사용자가 소프트웨어를 어떤 관점에서 보고 어떻게 조작하는 중점을 둔다.

### UI 구성 요소
- 시각적 디자인 : 색상, 타이포그래피, 레이아웃, 아이콘 등 사용자가 화면에 보이는 모든 요소  
- 상호작용 디자인 : 버튼 클릭, 스와이프 등 사용자의 조작에 대한 반응  
- 정보 구조 : 콘텐츠의 구성 및 요소

---

## UX(User Experience) : 사용자 경험
- 사용자가 소프트웨어를 사용하며 느끼는 전체적인 경험을 의미한다.  
- UX는 사용자가 소프트웨어를 통해 무엇을 얻고 느끼는 지에 중점을 둔다.

### UX 구성 요소
- 사용성 : 사용자가 소프트웨어를 얼마나 쉽게 이해할 수 있는지 정도  
- 유용성 : 소프트웨어가 사용자의 요구사항을 얼마나 충족하는지 정도  
- 매력도 : 소프트웨어거 사용자에게 얼마나 매력적인지 정도  
- 접근성 : 다양한 사용자가 소프트웨어를 얼마나 쉽게 이해할 수 있는지 정도

---

## UI / UX 의 관계
- 서로를 밀접하게 관련이 있긴 하다
- UX를 위해서는 UI가 필수지만, 좋은 UI가 항상 좋은 UX를 보장하지 않는다.
- UI와 UX는 서로 보완하면서 사용자의 만족도를 높이는데 기여한다.


## UI / UI 표준
- 개발자마다 서로 다른 방식으로 화면을 설계하면 사용자가 혼란을 느낄 수 있다.
- 이러한 문제를 줄이기 위해 공통적으로 참고할 수 있는 기준을 마련한 것이다.

### UI / UX 표준이 필요한 이유
- 일관성 있는 사용자 경험 제공
- 사용성 향상
- 개발 품질 향상
- 유지보수의 용이함
- 접근성 향상

---

### ISO (International Organization for Standardzation) : 국제 표준화 기구
- 전 세계 국가들이 공통적으로 사용할 수 있는 기술 표준과 품질 기준을 개발하고 관리한다.
```
  ISO 9421-210
      사용자 중심 설계를 체계적으로 수행할 수 있도록 국제적으로 제정된 표준이다.
      사용자와 컴퓨터가 상호작용하는 시스템을 설계할 때 사용자의 요구사항과 경험을 중심으로
      설계하도록 안내하는 국제 표준 기구이다.
```
### 사용자 중심 설계의 6대 원칙
1. 사용자를 명확하게 이해하고 설계한다.
2. 사용자를 설계 과정 전반에 참여시킨다.
3. 사용자 평가를 기반으로 설계한다.
4. 반복적인 설계 과정을 수행한다.
5. 사용자 경험 전체를 고려한다.
6. 다양한 전문가가 협력하여 설계한다.

- 사용자 중심 설계는 요구사항 분석 -> 프토토타입 제작 -> 사용자 평가 -> 개선 -> 재평가 단계를 거친다.
