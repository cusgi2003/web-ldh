객체지향의 4가지 개념

  캡슐화

    데이터를 보호하는 개념
    필드를 직접 접근하지 못하게 한다
    제공된 메서드를 통해서만 접근할 수 있다

  상속

    기존 클래스의 내용을 몰려받는 기능(필드,메서드)
    코드를 재사용할 때 좋음

  다형성

    하나의 타입으로 여러 객체를 다룰 수 있는 성질
    각 객체마다 성능이 다르다

  추상화

    복잡한 실체에서 공통적인 특징만 추출하여 단순화 하는 과정 


변수명을 지을 때 지켜야하는 규칙

    1. 숫자가 먼저 들어가면 안된다
    2. 영어 대소문자를 구분한다
    3. 일반적으로 영어소문자로 시작한다
    4. _, $를 제외하고 특수기호가 포함될 수 없다
    5. 예약어 금지 (switch, while 등)
    6. 한글은 사용하지 말 것


오버라이딩과 오버로딩

  오버라이딩

    부모 클래스에 존재하는 메서드를 자식클래스에 다시 정의(상속 필요)
    부모 클래스의 동작을 자식 클래스에 맞춰 변경(재정의)
    

  오버로딩    

    한 클래스 내에서 같은 이름의 메서드를 여러개 정의
    같은 동작을 하는 메서드에 다양한 매개변수를 제공하여 유연성 증대


상속이란

  이미 만들어진 클래스의 필드와 메서드를 다른 클래스가 물려받는 것
  다중상속이 불가
  중복코드를 줄이고 계층 구조를 형성하여 관리를 편하게 함


클래스 설계 / 객체
  필드, 생성자, 메서드

  Class Car{
    // 1. 필드
    String model;

    // 2. 생성자
    public Car(String model){
        this.model = model;
    }

    // 3. 메서드
    void drive() {
        System.out.println(model + "이(가) 달립니다.");
    }

    // 객체 생성
    Car myCar = new Car("소나타");
    myCar.drive();
