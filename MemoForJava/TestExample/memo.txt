객체지향의 4가지 개념

  캡슐화

    데이터를 보호하는 개념
    필드를 직접 접근하지 못하게 한다
    제공된 메서드를 통해서만 접근할 수 있다

  상속

    기존 클래스의 내용을 몰려받는 기능(필드,메서드)
    코드를 재사용할 때 좋음

  다형성

    상속 + 오버라이딩 + 타입변환
    하나의 타입으로 여러 객체를 다룰 수 있는 성질
    각 객체마다 성능이 다르다

  추상화

    중요한 공통점만 뽑아내고, 불필요한 세부부분은 감추는 것


변수명을 지을 때 지켜야하는 규칙

    1. 숫자가 먼저 들어가면 안된다
    2. 영어 대소문자를 구분한다
    3. 일반적으로 영어소문자로 시작한다
    4. _, $를 제외하고 특수기호가 포함될 수 없다
    5. 예약어 금지 (switch, while 등)
    6. 한글은 사용하지 말 것


오버라이딩과 오버로딩

  오버라이딩
    
    한 클래스 내에서 같은 이름의 메서드를 여러개 정의
    같은 동작을 하는 메서드에 다양한 매개변수를 제공하여 유연성 증대

  오버로딩    
  
    부모 클래스에 존재하는 메서드를 자식클래스에 다시 정의(상속 필요)
    부모 클래스의 동작을 자식 클래스에 맞춰 변경(재정의)


상속이란

  관련있는 것들끼리만 묶는 것
  공통적인 속성이나 동작을 부모쪽에 몰아놓는 것
  다중상속이 불가

  seald : 봉인된 클래스
  permit : 허용한 클래스만 상속


클래스 설계 / 객체
