# 논리 데이터저장소 확인하기
## 학습 1. 논리 데이터저장소 확인하기
### 1. 일반적인 시스템 개발 절차
- 개념모델링을 통해개발 범위를 파악하고, 업무 중심의 분석(논리 데이터 모델링, 분석모델링)
- 개발하고자 하는 환경을 고려한 설계(물리 데이터 모델링, 설계 모델링)
- 개발(데이터베이스 구축, 애플리케이션 개발)
---
### 2. 데이터 모델링 개요
- 정의 : 기업의 정보 구조를 실체와 관계 중심으로 명확하고 체계적으로  표현하여 문서화하는 기법
  ```
    실체(Entity) : 저장하고 관리해야 하는 실제 대상이나 개념
                   실체는 속성을 가진다
                   여러 개 존재 가능, 속성을 가짐, 구분 가능해야 함
  ```  
  ```
    관계(Relation) : 실체와 실체가 서로 어떤 연결을 가지는지 표현하는 것
                     관계는 보통 동사 형태를 사용한다
    ㄴ 1:1관계
    ㄴ 1:N관계
    ㄴ N:M관계
  ```
### 3. 데이터 모델링 절차
- 개념 모델링, 논리 모델링, 물리 모델링을 통해 데이터베이스를 구축하는 일련의 절차
```
  개념 데이터 모델링
        주요 엔티티타입, 기본 속성, 관계, 주요 업무기능 등 포함

  논리 데이터 모델링
        업무영역의 업무 데이터 및 규칙을 구체적으로 표현한 모델
        엔티티타입, 속성, 관계, 프로세스 등을 포함
        정규화 하여 모델링
        규칙과 관계를 완전하고 정확하게 표현

  물리 데이터 모델링
        시스템의 설계적 및 정보 요건을 정확하고 완전하게 표현한 모델
        데이터베이스 생성을 위한 물리 구조로 변환
```
---

### 4. 논리 데이터 모델링 개요
1. 데이터베이스 개발 과정의 첫 단계로 전략수리비 및 분석 단계에서 실시
2. 데이터 구조에 대한 논리적 정의 단계로서 현재 사용중인 양식, 문서, 장표를 중심으로
   자료 항목을 추출하여 추출된 엔티티와 속성들의 관계를 구조적으로 정의하는 단계
